[REBALANCE] REBALANCING WORKFLOW (Every 63 Trading Days):

STEP 1: IDENTIFY UNDERPERFORMERS
    - Metric: Average daily returns over 252-day lookback window (1 year)
    - Selection: Bottom 20% of current portfolio sorted by returns
    - Result: ~3 stocks flagged as underperforming

STEP 2: QUICK SECTOR-BASED REPLACEMENT
    - Action: For each underperformer, find best performer in SAME sector
    - Metric: Highest average daily return in that sector
    - Constraint: Only replace with available stocks (not already selected)
    - Purpose: Maintain sector diversification while swapping out weak stocks
    - Result: 'Seeded' portfolio with sector-balanced replacements

STEP 3: BUILD QUBO MATRIX
    - Input Data:
       * Expected returns (mu) - mean of returns
       * Covariance matrix (Sigma) - asset correlations
       * Downside volatility - penalty for negative deviations
    - QUBO Terms:
       * mu_i x w_i: Maximize expected returns
       * Sigma: Minimize portfolio variance (risk)
       * Downside risk: Penalize volatility
       * lambda_card x (|S| - K)^2: Cardinality constraint (exactly K assets)
       * gamma_sector: Sector concentration penalty

STEP 4: RUN QUANTUM ANNEALING
    - Candidate Pool: Top 100 stocks by Sharpe-like score
    - Algorithm: Simulated Annealing (quantum-inspired)
    - Objective: Select K=15 assets that maximize return-risk trade-off
    - Output: 15 assets optimized via QUBO formulation

STEP 5: OPTIMIZE WEIGHTS FOR SHARPE RATIO
    - Problem: Given 15 selected stocks, allocate optimal weights
    - Metric: Sharpe Ratio = (E[R] - Rf) / sigma
       * E[R]: Expected portfolio return
       * Rf: Risk-free rate (5%)
       * sigma: Portfolio volatility
    - Constraint: Max 12% per asset, optimal risk-adjusted allocation
    - Result: Portfolio weights that maximize excess return per unit risk

STEP 6: MERGE & FINALIZE
    - Seeded Portfolio: 13-15 stocks (from sector replacements)
    - Annealing Portfolio: 15 stocks (from QUBO + quantum annealing)
    - Merge: Combine both, remove duplicates, keep top K=15
    - Apply Transaction Costs: Penalize high portfolio turnover
    - Update Weights: Use merged assets with Sharpe-optimized weights

[STEP 1] DATA PREPARATION
   - Input: Historical price time series (15 years, 680 assets)
   - Clean prices: Remove missing data, impute, handle edge cases
   - Compute daily returns: r_t,i = P_t,i / P_{t-1,i} - 1
   - Annualize: mu_i = 252 * mean(r), Sigma = 252 * Cov(r)
   - Output: mu (expected returns), Sigma (covariance matrix)

[STEP 2] CONSTANTS & PARAMETERS
   - Risk weight (q): 0.5 (controls covariance penalty)
   - Downside penalty (beta): 0.3 (penalizes crash-prone stocks)
   - Portfolio size (K): 15 assets
   - Risk-free rate (r_f): 5.0%
   - Lambda (cardinality): 50.00 (ensures exactly K assets)
   - Gamma (sector): 5.00 (limits sector concentration)

[STEP 3] DOWNSIDE RISK CALCULATION
   - For each stock: DD_i = sqrt(252 * Var(negative_returns))
   - Penalizes stocks with high crash vulnerability
   - Used in QUBO matrix diagonal: beta * DD_i

[STEP 4] QUBO FORMULATION & MATRIX CONSTRUCTION
   - Objective: min E(x) = x^T Q x
   - Diagonal terms: Q_ii = q*Sigma_ii + mu_i - beta*DD_i - lambda(1-2K)
   - Off-diagonal: Q_ij = q*Sigma_ij + lambda + gamma*A_ij
   - Ensures: exactly K assets selected (cardinality)
   - Ensures: sector diversification (at most 4 per sector)

[STEP 5] SIMULATED ANNEALING (QUANTUM-INSPIRED OPTIMIZATION)
   - Initialize: random binary vector x_i ~ Bernoulli(0.5)
   - Cool: T_k = T_0 * alpha^k (from 2.0 to 0.005)
   - Search: flip bits to minimize E(x)
   - Accept: Metropolis rule with temperature-controlled probability
   - Output: x* = argmin E(x), selected assets = {i: x*_i = 1}
   - Result: 15 assets optimized for return-risk trade-off

[STEP 6] WEIGHT OPTIMIZATION (SHARPE RATIO MAXIMIZATION)
   - Objective: max (w^T mu - r_f) / sqrt(w^T Sigma w)
   - Constraints: SUM(w_i) = 1, 0 <= w_i <= 0.12 (max weight)
   - Solver: SLSQP (sequential least-squares programming)
   - Output: Optimal weights for 15 selected assets

[STEP 7] REAL-WORLD EXECUTION LAYER
   ========================================
   CRITICAL INNOVATIONS (Added to Bridge Theory <-> Practice)
   ========================================
   7.1 DISCRETE SHARE ALLOCATION
       - allocation_i = w_i * Budget
       - shares_i = floor(allocation_i / price_i)
       - invested_i = shares_i * price_i
       - Result: Whole shares only, respects budget
   7.2 CASH HANDLING
       - cash = Budget - SUM(invested_i)
       - Uninvested funds earn risk-free rate
       - Cash considered as portfolio component
   7.3 EFFECTIVE WEIGHTS
       - w_i^actual = (shares_i * price_i) / Budget
       - w_cash = cash / Budget
       - Sum = 1.0 (portfolio fully allocated)
   7.4 PORTFOLIO RETURNS
       - R_t = SUM(w_i^actual * r_{t,i}) + w_cash * r_f
       - Asset return + risk-free contribution
       - Daily computation ensures accurate cash drag
   7.5 TRANSACTION COSTS
       - turnover = SUM|w_i^new - w_i^old| / 2
       - cost = turnover * 0.2% per rebalance
       - Penalizes high portfolio turnover

[STEP 8] QUARTERLY REBALANCING
   - Cadence: Every 63 trading days (~quarter)
   - Lookback: 252-day window identifies underperformers
   - Strategy: Replace bottom 20% with same-sector performers
   - Optimization: Run QUBO + SLSQP to update weights
   - Costs: Apply transaction costs based on new turnover

[STEP 9] METRICS COMPUTATION
   - Total Return: (Final_Value / Initial) - 1
   - Annualized Return: exp(mean_log_return * 252) - 1
   - Volatility: std(log_returns) * sqrt(252)
   - Sharpe Ratio: (Annual_Return - r_f) / Volatility
   - Max Drawdown: min((Value_t - MaxValue) / MaxValue)
   - VaR_95: 5th percentile of daily returns
